# Plan First 工作流

## 核心理念

> "计划的质量决定了AI输出的上限。" —— Boris Cherny

## 流程

### Step 1: 进入Plan Mode

按两次 `Shift+Tab` 进入Plan Mode，或使用此命令启动。

### Step 2: 明确目标

描述你要实现的目标：

**用户目标**：$ARGUMENTS

### Step 3: 上下文收集

使用 `mcp__ace-tool__search_context` 搜索相关代码：

```
搜索与目标相关的现有实现、接口定义、相关模块。
```

### Step 4: 方案讨论

与Claude讨论实现方案：
- 可选的技术路线
- 各方案的优缺点
- 对现有代码的影响

**可调用**：
- `architect` agent - 架构决策
- `ui-expert` agent - UI设计决策

### Step 5: 计划确认

生成详细的实施计划：

```markdown
## 实施计划

### 1. [第一步]
- 修改文件: xxx
- 具体改动: xxx

### 2. [第二步]
...

### 验证方式
- 测试命令: xxx
- 预期结果: xxx
```

### Step 6: 获得批准

向用户展示计划，询问：

> **是否按此计划执行？(Y/N)**

等待用户确认后再进入执行阶段。

### Step 7: 切换到执行模式

用户确认后：
1. 退出Plan Mode
2. 按计划逐步执行
3. 每步完成后更新Todo状态
4. 完成后运行 `/mmo:verify`

## 适用场景

- 新功能开发
- 重大重构
- 架构变更
- 不确定最佳实现方式时

## 不适用场景

- 简单bug修复
- 单行代码改动
- 明确的小任务
